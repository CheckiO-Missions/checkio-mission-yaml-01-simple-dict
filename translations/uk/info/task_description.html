Тобі відома така мову розмітки, як YAML? Це дружній формат представлення даних. Насправді, він настільки дружній, що і люди, і програми можуть його читати досить добре. Ти можеш дізнатися більше про формат за цим посиланням.

YAML це текст, і тобі потрібно перетворити його на об'єкт. Але я не прошу тебе реалізувати весь стандарт YAML, ми реалізуємо його поетапно.

Перший крок - це перетворення пар ключів-значень. Ключем може бути будь-який рядок, що складається з латинських літер та чисел. Значення може бути однолінійний рядок (який складається з пробілів, латинських літер та чисел) або числа ({% if is_js %}Number{% else %}int{% endif %}).

Я покажу кілька прикладів:

name: Alex
age: 12

Перетворюється на {% if is_js %}об'єкт{% else %}словник{% endif %}.

{ 
  "name": "Alex",
  "age": 12
}

Зауваж, що число автоматично отримує тип {% if is_js %}Number{% else %}int{% endif %}.

Інший приклад показує, що рядок може містити пробіли.

name: Alex Fox
age: 12

class: 12b

Буде конвертовано у наступний {% if is_js %}об'єкт{% else %}словник{% endif %}.

{
  "age": 12, 
  "name": "Alex Fox", 
  "class": "12b"
}

Зверни увагу на кілька речей. Між рядком "age" і рядком "class" є порожній рядок, який не заважає конвертації. "class" починається з числа, але має літери, а це означає, що його не можна перетворити на число, тому його тип залишається {% if is_js %}String{% else %}str{% endif %}.

Вхідні дані: Форматований рядок.

Вихідні дані: об'єкт ({% if is_js %}тип Object{% else %}словник{% endif %}).
