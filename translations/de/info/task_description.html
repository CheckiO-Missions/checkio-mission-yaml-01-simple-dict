Haben Sie schon einmal von einer Auszeichnungssprache wie YAML gehört? Es handelt sich dabei um ein benutzerfreundliches Format zur Serialisierung von Daten. Es ist sogar so freundlich, dass sowohl Menschen als auch Programme es gut lesen können. Sie können mit dem Standard herumspielen, indem Sie this link folgen.

YAML ist ein Text, den Sie in ein Objekt umwandeln müssen. Aber ich verlange nicht von Ihnen, dass Sie den gesamten YAML-Standard implementieren, wir werden ihn Schritt für Schritt implementieren.

Der erste Schritt ist die Umwandlung von Schlüsseln in Werte. Der Schlüssel kann eine beliebige Zeichenkette sein, die aus lateinischen Buchstaben und Zahlen besteht. Der Wert kann eine einzeilige Zeichenkette (die aus Leerzeichen, lateinischen Buchstaben und Zahlen besteht) oder eine Zahl sein ({% if is_js %}Number{% else %}int{% endif %}).

Ich werde einige Beispiele zeigen:

name: Alex
age: 12

Umgewandelt in {% if is_js %}ein Objekt{% else %}ein Wörterbuch{% endif %}.

{ 
  "name": "Alex",
  "age": 12
}

Beachten Sie, dass die Zahl automatisch den Typ {% if is_js %}Number{% else %}int{% endif %} erhält.

Ein weiteres Beispiel zeigt, dass die Zeichenkette Leerzeichen enthalten kann.

name: Alex Fox
age: 12

class: 12b

Wird in das nächste {% if is_js %}Objekt{% else %}Wörterbuch{% endif %} umgewandelt.

{
  "age": 12, 
  "name": "Alex Fox", 
  "class": "12b"
}

Achten Sie auf ein paar Dinge. Zwischen der Zeichenkette "age" und der Zeichenkette "class" befindet sich eine leere Zeichenkette, die das Parsen nicht behindert. Die Klasse beginnt mit Zahlen, hat aber Buchstaben, was bedeutet, dass sie nicht in Zahlen umgewandelt werden kann, so dass ihr Typ {% if is_js %}String{% else %}str{% endif %} bleibt.

Eingabe: Eine Formatzeichenkette.

Ausgabe: Ein Objekt ({% if is_js %}Objekttyp{% else %}Wörterbuch{% endif %}).

{{init_code_tmpl}}
